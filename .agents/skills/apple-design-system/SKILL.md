# Apple design system

Start from semantic tokens, neutral defaults, and native SwiftUI behavior. Check Dynamic Type, appearance, increased contrast, Reduce Motion, Differentiate Without Color, VoiceOver, macOS keyboard/pointer, and iOS touch targets.

Inputs: shared token or component change. Output: neutral documented behavior, focused tests/examples, and gallery/build verification. Do not introduce branding or implement a planning-only request.
