# SwiftUI component author

Accept the component responsibility, consumers, platforms, and configuration. Use semantic configuration and a small public API; keep state ownership explicit; use native controls; support Dynamic Type, appearance, VoiceOver, keyboard access, and platform interaction. Add DocC documentation, a self-contained preview/example, focused tests, and run the relevant build.
