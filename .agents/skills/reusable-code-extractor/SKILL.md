# Reusable code extractor

Inspect the working application behavior and tests. Identify and remove project terminology, domain models, branding, and hidden assumptions. Design the smallest reusable API, preserve behavior with tests, add a generic example, verify the application integration still works, and stop if fewer than two plausible consumers remain.
