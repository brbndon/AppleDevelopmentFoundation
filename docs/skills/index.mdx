# Skills inventory

**Source of truth:** `.agents/skills/manifest.json` (install/verify) and `.agents/skills/README.md` (use / do-not-use). Router catalog: `apple-development-foundation/master-skill.json`.

This table was last reviewed against `manifest.json` on **2026-07-17** (15 skills). When the manifest or skill purposes change, update this page in the same change set — do not leave agents on a stale inventory.
:::

## Inventory

| Skill | Use for | Do not use for |
| --- | --- | --- |
| `apple-development-foundation` | Reference-first child-skill routing; explicit foundation audits | Replacing child instructions; implied audits/verification |
| `codex-bootstrap` | New iOS/macOS SwiftUI projects in the **consumer** workspace | One-off screens; archived package work in this repo |
| `apple-platform-planner` | Planning a platform feature | Implementation-only requests |
| `swiftui-tab-navigation` | App-level tabs / system Liquid Glass tab chrome | Page-style tabs, segmented controls, business screens |
| `swiftui-component-author` | Reusable SwiftUI components | App-specific screen copy |
| `apple-design-system` | Design-system tokens and components | Application branding |
| `ios-macos-platform-adaptation` | Divergent iOS vs macOS behavior | Shared identical behavior |
| `swift-package-module-author` | Independently importable SPM module in the active workspace | App-local feature; archived package here |
| `reusable-code-extractor` | Proven app code → consumer shared module | Speculative abstraction; landing code in this repo |
| `swift-concurrency-review` | async/await, actors, cancellation | Unrelated styling |
| `apple-accessibility-review` | Reusable UI accessibility | Non-UI services |
| `apple-security-privacy-review` | Files, logging, permissions, sensitive data | Visual-only polish |
| `swift-testing-verification` | Verification after meaningful code changes | Planning-only work |
| `maestro-apple-app-testing` | End-to-end Maestro UI testing | Non-testing product implementation |
| `codex-skill-maintainer` | Creating/updating skills in this repo | Ordinary product code changes |

Deep procedure: `.agents/skills/<name>/SKILL.md` — prefer reading the skill over forking long instructions into this site.
